Docker Containers:

Docker Containerization includes all dependencies (frameworks, libraries, etc) required to run an application in an efficient and bug free manner. With Docker containers, applications can work efficiently in different computer environments. The benefits of Docker container show up in various places. Since the Docker container is lightweight, it can be easily deplpoyed and executed on other computer environments regardless of the host operated system of configurations. Containers take up less disk space and are relatively faster in executing applications. Containers run applications in isolation and share the operation system kernel with other containers.

Docker:How to build and run an image as a container

  • Step 1: Open Git Bash and run the following command: docker run -d -p 80:80 docker/your-image
  • Step 2: Open your Docker Dashboard and you should see the container you created running!

Docker images:

A Docker image is made up of a collection of files that bundle together all the essentials – such as installations, application code, and dependencies – required to configure a fully operational container environment. You can create a docker image with two different methods.
  • Interactive: By running a container from an existing Docker image, manually changing that container environment through a series of live steps, and saving the resulting state as a new image.
  • Dockerfile: By constructing a plain-text file, known as a Dockerfile, which provides the specifications for creating a Docker image.

Docker commands:

  • docker-version:This command is used to get the currently installed version of docker
  • docker pull: This command is used to pull images from the docker repository(hub.docker.com)
  • docker run -it -d : This command is used to create a container from an image
  • docker ps: This command is used to list the running containers
  • docker ps-a: This command is used to show all the running and exited containers
  • docker exec -it : This command is used to access the running container
  • docker compose up --build: This command aggregates the output of each container

Step by Step, How to start the project via Terminal:

To start project via terminal with terminal commands without pycharm you'll utilize docker compose up --build. The docker-compose up command aggregates the output of each container. When the command exits, all containers are stopped.
