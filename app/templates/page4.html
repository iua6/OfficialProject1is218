Project setup for github:Step by Step

For Continuous Integration and Deployment
  • Clone this repo to your local
  • Create a new repo on your own Github account
  • Change the origin by replacing it with your own new repo. (Do not add a readme or any other file while creating the repository)
  • Create an account with Heroku, create an app for production and an app for development
  • Create a new repo in Docker hub
The following steps will automatically trigger the Production workflow that is in the prod.yml after any changed is committed on Github.

Code review using dev server:

To reveiw code before merging pull requests you will navigate to 'actions' then on the left you should see all your Workflows including the Development workflow and the details.

How to deploy master to production server: Step by Step

  • Create a bare repo: Make sure you are allowed to write to it
  • Create a file to recieve
  • Configure your local repo to push to the server
  • The goal is when you run git push, deploy the latest master branch to the server
