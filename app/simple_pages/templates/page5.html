The Arrange, Act, and Assert (AAA) Pattern



What it is:

The AAA pattern is a test-structure pattern. It divides each test into three sections – Arrange, Act, and Assert – with each section serving as a step toward the next. The arrange phase is responsible for configuring the test's input values. The principal function being tested is prompted by the act step. Finally, the assert step ensures that the function's output matches what was intended.

AAA Testing with python Tutorial:

  1. Create a 'Testfixture' class that will accessible to every test.
  2. Create a single method called 'Test' with three parameters that are all functions.
  3. Use anonymous functions to pass functions as parameters
  4. The anonymous functions are called in a test method
  5. If the signatures of the functions you create match the signatures of the test method’s parameters, you can use the test method in different test cases

Importance:

Overall, the AAA pattern is just meant to be used as a guideline. It's a framework, or a technique of thinking about and organizing your tests such that they're easy to understand. There is no documentation included in any package or downloaded tool. Instead, it is up to you, the author, to decide how to best produce and use it. Creating functions for each step in the pattern has proven to be one of the more adaptable options. Functions are similar to building blocks in that they may be reused in multiple designs. In our example, you may reuse distinct functions across several test cases.
